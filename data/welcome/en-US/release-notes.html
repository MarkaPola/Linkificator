History of changes

Version 1.6.3

Enhancements and bug fixes

  • Enhances widget handling in multi-windows mode.

Limitations

  • Due to new add-on packaging, minimal version supported is now Firefox 21.

Version 1.6.2

Enhancements and bug fixes

  • Optimizes document processing.

Limitations

  • Due to new add-on packaging, minimal version supported is now Firefox 21.

Version 1.6.1

Enhancements and bug fixes

  • It is now possible to deactivate automatic processing of dynamic changes (see 'Advanced settins', 'Links' tab).
  • Optimizes automatic processing to avoid potential lags.

Limitations

  • Due to new add-on packaging, minimal version supported is now Firefox 21.

Version 1.6.0

New features

  • The dynamic changes of current page are now detected and automatically processed.

Limitations

  • Due to new add-on packaging, minimal version supported is now Firefox 21.

Version 1.5.3

Enhancements and bug fixes

  • Fix erroneous handling of inline elements.

Limitations

  • Per window private browsing mode on Firefox 20 is not compatible with SDK panels feature. This feature is used by Linkificator for context menu.
    So, on Firefox 20, Linkificator context menu is deactivated. It is strongly recommended to upgrade to Firefox 21 and above to have a fully operational Linkificator.

Version 1.5.2

Enhancements and bug fixes

  • By default, to reduce even more the false positives, the URLs containing only the authority (i.e. host:port) are no longer linkified, even if they contain a valid top level domain.
    The option 'Advanced Options->Links->Linkify URL containing the alone authority' enables to activate this treatment.
  • Enhances matching of URIs.
  • Speed-up processing when Top Level Domains are used.

Limitations

  • Per window private browsing mode on Firefox 20 is not compatible with SDK panels feature. This feature is used by Linkificator for context menu.
    So, on Firefox 20, Linkificator context menu is deactivated. It is strongly recommended to upgrade to Firefox 21 and above to have a fully operational Linkificator.

Version 1.5.1

Enhancements and bug fixes

  • Enhances identification of start of URIs.
  • Enhances Top Level Domain matching.

Limitations

  • Per window private browsing mode on Firefox 20 is not compatible with SDK panels feature. This feature is used by Linkificator for context menu.
    So, on Firefox 20, Linkificator context menu is deactivated. It is strongly recommended to upgrade to Firefox 21 and above to have a fully operational Linkificator.

Version 1.5.0

New features

  • To reduce false positives, URI are now recognized only if the domain part is terminated with a known top level domain.
    It is possible to de-activate this mode (and so, reverting to the behavior of the previous version) through 'Advanced settings', tab 'Links'.

Limitations

  • Per window private browsing mode on Firefox 20 is not compatible with SDK panels feature. This feature is used by Linkificator for context menu.
    So, on Firefox 20, Linkificator context menu is deactivated. It is strongly recommended to upgrade to Firefox 21 and above to have a fully operational Linkificator.

Version 1.4.2

Enhancements and bug fixes

  • Speeds up the processing of inline tags to reduce drastically the potential freezes of Firefox.
  • For a better compatibility with 'Pale Moon' browser, changes slightly the checking of version of Firefox to disable the panel on version 20.

Limitations

  • Per window private browsing mode on Firefox 20 is not compatible with SDK panels feature. This feature is used by Linkificator for context menu.
    So, on Firefox 20, Linkificator context menu is deactivated. It is strongly recommended to upgrade to Firefox 21 and above to have a fully operational Linkificator.

Version 1.4.1

Enhancements and bug fixes

  • Enhance page history navigation handling (Back/Forward buttons).

Limitations

  • Per window private browsing mode on Firefox 20 is not compatible with SDK panels feature. This feature is used by Linkificator for context menu.
    So, on Firefox 20, Linkificator context menu is deactivated. It is strongly recommended to upgrade to Firefox 21 and above to have a fully operational Linkificator.

Version 1.4.0

New features

  • Left click is now configurable. As a result, menu can now be assigned to any mouse button.
    By default, for compatibility purpose, menu display is binded to left click.

Enhancements and bug fixes

  • Per window private browsing mode is now fully supported but have some restrictions on Firefox 20 due to SDK limitations specifically on this version.

Limitations

  • Per window private browsing mode on Firefox 20 is not compatible with SDK panels feature. This feature is used by Linkificator for context menu.
    So, on Firefox 20, Linkificator context menu is deactivated. It is strongly recommended to upgrade to Firefox 21 and above to have a fully operational Linkificator.

Version 1.3.0

New features

  • Page context menu integration: it is possible to update links through page context menu (customizable).
  • Middle click on Widget is now configurable. No action is binded by default.

Enhancements and bug fixes

  • Change behavior of disabled state : linkificator actions are now reverted when Linkificator is disabled.
  • Fix erroneous handling of status tooltip.

Limitations

  • Per window private browsing mode (Firefox 20+) is not supported due to an Add-on SDK bug (see 816257).
    The support will be added as soon as SDK bug is fixed (planned for SDK 1.15 version).

Version 1.2.0

New features

  • Possibility to toggle processing of the current page.
    This action can be launched through widget menu (Exclude/Include) or the hot key control-shift-x (customizable).

Enhancements and bug fixes

  • Enhance anchor handling when inline elements are involved.
  • Add tag <wbr> to inline elements.

Version 1.1.1

New features

  • Possibility to update the current page without reloading it.
    This action can be launched by a right click on the Linkificator widget or through the hot key control-shift-u (customizable).
  • Left click on the widget now display a menu for an easy access to the most common operations as well as the options page.
  • An action can be directly executed with a right click on the widget. This action is customizable (see Options).

Enhancements and bug fixes

  • Fix custom rule checkbox handling on Mac OS.

Version 1.0.3

Enhancements and bug fixes

  • Enable custom rules edition on Mac OS.
  • Enhance statistics handling.
  • Fix erroneous inline elements processing.

Version 1.0.2

Enhancements and bug fixes

  • Enhance URLs parsing.
  • Reduce resources consumption and potential UI freezes.

Version 1.0.1

Enhancements and bug fixes

  • Tag <span> is now handled has an inline element.
  • Enhance localization messages.

Version 1.0.0

New features

  • Inline elements, like <big> or <strong>, are now recognized and correctly handled during URLs parsing.
  • Existing pages are now automatically parsed when add-on is installed or re-enabled. It is no longer required to manually reload the page.

Enhancements and bug fixes

  • Multi-frames documents are now correctly handled in all cases.
  • plain text documents are now recognized and parsed.
