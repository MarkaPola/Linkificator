Linkificator 1.5.2

Nouvelle version installée.

Merci d'avoir choisi Linkificator.
Pour de plus amples informations, visitez la page du module.

Historique des changements

Version 1.5.2

Améliorations et correction de bogues

  • Par défaut, pour réduire encore les faux positifs, les URLs ne contenant que l'autorité (i.e. serveur:port) ne sont plus traités, même si elles contiennent un domaine de premier niveau valide.
    L'option 'Réglages Avancés->Liens->Traiter les URL contenant uniquement l'autorité' permet d'activer ce traitement.
  • Amélioration de la détection des URIs.
  • Amélioration de la vitesse de traitement lorsque les domaines de premier niveau sont utilisés.

Limitations

  • La navigation privée sur Firefox 20 n'est pas compatible avec la fonctionnalité des panels du SDK. Cette fonctionnalité est nécessaire à la gestion du menu contextuel de Linkificator.
    En conséquence, sur Firefox 20, le menu contextuel de Linkificator est désactivé. Il est donc fortement recommandé de mettre à jour Firefox vers la version 21 ou suivantes pour avoir un fonctionnel complet de Linkificator.

Améliorations et correction de bogues

  • Amélioration de la détection du début des URIs.
  • Amélioration de la reconnaissance des domaines de premier niveau.

Limitations

    En conséquence, sur Firefox 20, le menu contextuel de Linkificator est désactivé. Il est donc fortement recommandé de mettre à jour Firefox vers la version 21 ou suivantes pour avoir un fonctionnel complet de Linkificator.

Version 1.5.0

Nouvelles fonctionnalités

  • Pour réduire les faux positifs, les URIs sont maintenant reconnus uniquement si le domaine se termine par un nom de domaine de premier niveau connu.
    Il est possible de désactiver ce mode (et donc de revenir au fonctionnement de la version précédente) via 'Réglages avancés', onglet 'Liens'.

Limitations

  • La navigation privée sur Firefox 20 n'est pas compatible avec la fonctionnalité des panels du SDK. Cette fonctionnalité est nécessaire à la gestion du menu contextuel de Linkificator.
    En conséquence, sur Firefox 20, le menu contextuel de Linkificator est désactivé. Il est donc fortement recommandé de mettre à jour Firefox vers la version 21 ou suivantes pour avoir un fonctionnel complet de Linkificator.

Version 1.4.2

Améliorations et correction de bogues

  • Amélioration de la vitesse de traitement des éléments en ligne qui permet de réduire considérablement les potentiels gels de Firefox.
  • Pour une meilleure compatibilité avec le navigateur 'Pale Moon', le test de version de Firefox a été légèrement changé pour permettre de désactiver le panel sur toutes les versions 20.

Limitations

Améliorations et correction de bogues

Version 1.4.0

Nouvelles fonctionnalités

  • Le clic gauche est maintenant configurable. En conséquence, le menu contextuel peut maintenant être assigné à n'importe quel bouton de la souris.
    Par défaut, pour rester compatible avec les versions précédentes, l'affichage du menu est assigné au clic gauche.

Améliorations et correction de bogues

  • La navigation privée par fenêtre est maintenant supportée mais avec certaines restrictions sur Firefox 20 en raison des limitations du SDK spécifiquement pour cette version.

Limitations

  • La navigation privée sur Firefox 20 n'est pas compatible avec la fonctionnalité des panels du SDK. Cette fonctionnalité est nécessaire à la gestion du menu contextuel de Linkificator.
    En conséquence, sur Firefox 20, le menu contextuel de Linkificator est désactivé. Il est donc fortement recommandé de mettre à jour Firefox vers la version 21 ou suivantes pour avoir un fonctionnel complet de Linkificator.

Version 1.3.0

Nouvelles fonctionnalités

  • Intégration au menu contextuel de page : Il est maintenant possible d'actualiser les liens via le menu contextuel de la page (configurable).
  • Le clic central sur le widget est maintenant configurable. Par défaut aucune action n'est exécutée.

Améliorations et correction de bogues

  • Changement du comportement de l'état désactivé : les actions de Linkificator sont annulées lorsque l'on désactive Linkificator.
  • Correction des messages erronés affichés par le tooltip.

Limitations

  • La navigation privée par fenêtre (Firefox 20+) n'est pas supportée due à un bogue du Add-on SDK (see 816257).
    Le support sera ajouté dés que le bogue du SDK sera corrigé (prévu pour la version 1.15 du SDK).

Version 1.2.0

Nouvelles fonctionnalités

  • Possibilité de désacriver/réactiver le traitement de la page courante.
    Cette action peut être lancée via le menu du widget (Exclure/Inclure) ou le raccourci clavier control-shift-x (configurable).

Améliorations et correction de bogues

  • Amélioration la génération du tag <a> dans le cas d'éléments en ligne.
  • Ajout du tag <wbr> aux éléments en ligne.

Version 1.1.1

Nouvelles fonctionnalités

  • Possibilité d'actualiser la page courante sans la recharger.
    Cette action peut être lancée via un clic droit sur le widget ou via le raccourci clavier control-shift-u (configurable).
  • Un clic gauche sur le widget affiche maintenant un menu pour un accès facilité aux opérations les plus courantes ainsi que la page d'options.
  • Une action peut être directement exécutée via un clic droit sur le widget. Cette action est configurable (voir la page d'options).

Améliorations et correction de bogues

  • Correction de la gestion de la checkbox pour les règles utilisateurs sur MacOS.

Version 1.0.3

Améliorations et correction de bogues

  • Ajout de la possibilité d'éditer les règles utilisateur sur Mac OS.
  • Amélioration de la gestion des statistiques.
  • Correction d'un traitement erroné des éléments en ligne.

Version 1.0.2

Améliorations et correction de bogues

  • Amélioration de l'analyse des URLs.
  • Réduction des besoins en ressources et de potentiels gels de l'interface graphique.

Version 1.0.1

Améliorations et correction de bogues

  • Le tag <span> est maintenant traité en tant qu'élément en ligne.
  • Amélioration de la localisation de certains messages.

Version 1.0.0

Nouvelles fonctionnalités

  • Les elements en ligne, tel que <big> ou <strong>, sont maintenant reconnus et correctement traités lors de l'analyse des URL.
  • Les Pages existantes sont maintenant automatiquement traitées lorsque l'extension est installée ou re-activée. Il n'est plus nécessaire de recharger la page manuellement.

Améliorations et correction de bogues

  • Les documents multi-frames sont maintenant correctement traités dans tous les cas.
  • Les documents texte sont reconnus et traités.
